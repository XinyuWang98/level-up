# /generate_gap_analysis 工作流

## 1. 触发与信息收集
当用户输入 `/generate_gap_analysis` 或类似请求时，Agent 需首先确认是否已提供**目标岗位的 JD (Job Description)** 以及**用户当前的背景信息**（或从上下文中提取/要求用户提供简历）。
- 如果信息不足，请礼貌地要求用户提供具体 JD 文本。

## 2. 分析与对比逻辑
1. **硬技能对标 (Hard Skills)**：将 JD 中的技术要求（如被动提及的工具库、特定的算法、因果推断方法等）与用户现状逐一映射。
2. **软技能/业务思维 (Soft Skills & Business Sense)**：对比岗位对“从0到1体系建设”、“横向跨部门协同”或“特定业务线的理解”的要求与用户的断层。
3. **优先级排序**：根据 JD 描述的侧重点（例如出现“精通”、“必备”的词汇优先度最高）为发现的 Gap 排列优先级。

## 3. 标准化输出格式
请严格按照以下 Markdown 格式输出分析结果：

### [目标岗位名称及公司] - 差距分析矩阵 (Gap Analysis)

| 技能/考察维度            | JD 期望水平    | 你的当前现状 | 差距描述 (Gap) | 优先级             | 下一步行动项 (Action Item)          |
| :----------------------- | :------------- | :----------- | :------------- | :----------------- | :---------------------------------- |
| **[维度例如：因果推断]** | [高度概括要求] | [你的现状]   | [具体缺什么]   | 🔴 高 / 🟡 中 / 🟢 低 | [例如：完成倾向性匹配实战 Notebook] |

### 阶段性突破建议 (Milestones)
- **短期 (1周内)**: [最急需弥补的硬核技能或知识盲区]
- **中期 (2-3周)**: [需要用实战项目或业务 case 积累的经验]
- **长期规划**: [职业视角的软技能与方法论沉淀]

## 4. 后续指引
在输出分析后，提示用户：“你可以将这份报告保存为 `03_职业规划/[你的名字]_Gap_Analysis.md`，并在后续学习中随时使用 `/update_senior_gap` 指令来更新进度！”
