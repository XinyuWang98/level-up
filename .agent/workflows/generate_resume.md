# 简历生成工作流

## ⛔ 封板规则 (Iron Rule)
1. **在修改任何简历之前**，必须先读取 `02_速查手册/docs/Resume_Tracker.md`。
2. 如果目标简历已在“封板清单”中（状态 = `✅ 已投递`），**禁止编辑**，改为更新基础版 `03_职业规划/Packaged_Resume.md`。

## 前置准备
1. 读取 `02_速查手册/docs/Resume_Tracker.md` 及 `03_职业规划/Packaged_Resume.md`。
2. 获取目标岗位 JD。

## 分析与生成阶段
1. **获取叙事映射 (Narrative Mapping)**：
   - 检查 `03_职业规划/Narrative_Mapping.md` 或 `03_职业规划/Profile.md`。
   - **动态感知**：如果不存在映射文件，Agent 应主动询问用户：“你的核心叙事转型方向是什么？（例如：金融转互联网、客服转专家数分）”，并根据回答动态执行升维，随后建议用户保存为映射文件。
2. **术语与场景映射**：根据映射规则，对简历中的初级/特定行业术语进行专业化翻译（如：转人工率 → 人工介入率）。
3. **数据一致性硬约束**：确保 9pp, 3.2pp, 30%方差缩减等核心数字在所有版本中绝对一致。
4. **格式规范**：纯中文标题（如 `## 核心优势`），技术名词加粗，每段经历必须包含 `> *背景：...*` 和 `**成果**`。

## 内容策略
- **高阶能力锚定**：即使 JD 未要求，也应保留因果推断、AB 实验平台建设等高阶描述，作为资深岗位的谈薪筹码。
- **隐私保护**：生成过程中严禁在文件名或正文中暴露用户真实住址或联系方式。

## 审阅与交付
1. 提交用户审阅，标注与基础版的核心差异。
2. 投递后同步更新 `02_速查手册/docs/Resume_Tracker.md`。
