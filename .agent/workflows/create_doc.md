---

## 流程步骤 (Process)

1.  **起飞前检查 (Pre-flight Check - Language)**:
    -   **核心规则**: 目标语言必须是 **简体中文 (Chinese Simplified)** (遵循全局规则 Rule 0)。
    -   **意图核对**: 确认这是技术文档编写还是翻译任务？无论哪种，输出必须是中文。

2.  **上下文加载 (Context Loading)**:
    -   读取相关文件以理解现有的文档结构和术语规范。
    -   如果是更新 `知识库 (knowledge_base)`，必须先读取 `00_教学课件/知识图谱.md` 或 `02_速查手册/docs/` 下的索引文件。

3.  **内容生成 (Content Generation)**:
    -   **中文优先**: 所有解释、注释、标题必须使用中文。
    -   **术语处理**: 仅保留核心英文术语 (如 DID, PSM, XGBoost)，但必须紧跟中文解释或定义。
    -   **格式规范**: 使用标准的 Markdown 格式 (H1/H2 标题, 列表, 代码块必须带中文注释)。

4.  **执行写入 (Execution)**:
    -   使用 `write_to_file` 或 `replace_file_content` 提交文件变更。

5.  **降落后验证 (Post-flight Verification)**:
    -   **回读文件**: 检查刚才写入的内容。
    -   **自我审查**:
        -   是否存在大段的英文未翻译？
        -   标题是否全是中文？
        -   代码注释是否为中文？
    -   **纠错**: 如果发现违反上述任何一条，**立即重写**，不要等待用户反馈。
