# 每日复盘工作流 (Dual-Perspective Daily Review)

此工作流用于生成一份绝对客观、公正的复盘报告。核心原则是将“教学质量”与“学习表现”解耦，拒绝单方面指责。

## 1. 核心原则
*   **Dual Perspective**: 必须明确区分 **Agent Self-Reflection** (导师做得怎么样) 和 **Student Performance** (学生学得怎么样)。
*   **No PUA**: 禁止将课程设计的缺失归咎于学生的基础。
*   **Metric-Driven**: 使用 S/A/B/C 评级，必须给出扣分/加分的具体理由。

## 2. 报告结构 (Report Structure)

报告必须包含以下三个模块：

### 🟢 Part 1: 学员高光与不足 (Student Performance)
*   **Critical Intuition (直觉)**: 学员是否指出了逻辑漏洞？(这是最高加分项)
*   **Code Proficiency (代码)**: 代码是否还在犯基础语法错误？(如果不影响逻辑，不作为主要扣分项)
*   **Business Sense (业务感)**: 提的问题是否切中业务痛点？

### 🔴 Part 2: 教学事故复盘 (Curriculum Audit)
*   **Missed Concepts**: 今天有没有漏讲的关键假设？(e.g., Parallel Trends)
*   **Reactive vs Proactive**: 是 Agent 主动讲的，还是学员问了才补的？(后者为教学失误)
*   **Realism Gap**: 案例是否过于理想化？(e.g., Perfect Separation)

### 🔵 Part 3: 明日改进计划 (Action Plan)
*   **Curriculum Fix**: Agent 需要去修改哪些教程文件？(Proactive)
*   **Student Drill**: 学员需要单纯练习哪些代码肌肉记忆？

## 3. 执行要求
1.  **文件名**: `XX_Daily_Review_YYYYMMDD.md`
2.  **tone**: 专业、冷静、平等。像 Senior 工程师给 Junior 做 Code Review，而不是老师训学生。
3.  **Strict Mode**: 如果教学有失误，必须在 Part 2 第一条承认。
