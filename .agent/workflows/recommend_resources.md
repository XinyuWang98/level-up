# /recommend_resources 工作流

## 1. 核心铁律 (CRITICAL RULES)
- **绝对禁止幻觉 (No Hallucinated URLs)**：Agent 绝不允许主观编造、推测或拼接 Bilibili、YouTube 或任何平台的视频链接。
- **强制搜索验证**：在给出任何链接之前，**必须**使用可用的联网搜索工具（Web Search / Search API）去检索真实存在的资源。
- **降级可用性**：如果在当前运行环境中 Agent **没有联网搜索能力**，必须诚实地告知用户，并改为输出**精确的“搜索关键词 (Search Queries)”**，由用户自行前往对应平台检索。

## 2. 执行逻辑
当用户请求关于某个难点（如 CUPED 方差缩减、DoWhy 框架解密）的视频或额外资料时：

**情景 A：拥有联网搜索能力**
1. 搜索目标主题的高质量解说（优先搜索 Bilibili 的中文资源，其次是 YouTube 的英文优质讲解）。
2. 核验链接后整理成带时间戳的推荐目录。

**情景 B：无联网能力（或搜索失败）**
1. 直接输出平台推荐与精准的检索词组串。

## 3. 标准化输出格式

### 🔍 推荐学习资源：[知识点名称]

**▶️ 视频讲解推荐**
（如果成功验真了链接，按以下格式输出；否则省略此块）
*   **[视频标题]**
    *   **链接**: [真实有效的URL]
    *   **时长**: [约 X 分钟]
    *   **核心看点**: [一句话总结，如：用大白话解释了为什么要用大盘数据作协变量]
    *   **时间戳建议**: [例如：从 12:30 开始讲解代码落地]

**🔑 高效检索建议 (手动搜索关键词)**
如果上面的链接失效，或者你想看更多维度的讲解，请直接复制以下关键词前往 B站 / YouTube 搜索：
- 核心原理解析：`"[知识点名称] 原理通俗解释"`
- 代码实战落地：`"Python [领域] [知识点] 代码教学"`
- 大厂面试面经：`"[知识点] 数据分析 面试题"`

## 4. 后续互动
询问用户看完视频后是否需要针对该知识点进行一场定制化的 `/mock_interview` 测试，或者生成一个简单的 `/generate_notebook` 供其代码练手。
