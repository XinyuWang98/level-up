# 投递进度更新工作流

## 触发条件
当用户提到"更新投递进度"、"帮我更新投递"、"记录一下投递"等关键词时自动触发。

## 需要同步更新的 3 个位置

### 1. 投递追踪表（数据源）
// turbo
- **文件**：`05_职业规划/Resume_Tracker.md`
- **操作**：在投递清单表格末尾追加新行，更新封板清单
- **状态规则**：
  - `✅ 已投递`：已成功通过平台投递简历
  - `💬 已沟通`：已在 BOSS 直聘等平台发消息，等待 HR 回复后投递
  - `📝 待投递`：已创建定制简历但尚未投递
  - `❌ 已拒`：收到拒信
  - `📞 约面`：收到面试邀请

### 2. 知识库面试准备页（展示层）
// turbo
- **文件**：`02_速查手册/docs/17_interview_prep.md`
- 需要同步更新以下 **3 个区域**：
  1. **5.1 投递总览甘特图**：追加新公司的 Gantt section
  2. **5.2 投递清单**：追加新行（与 Resume_Tracker.md 保持一致）
  3. **5.3 岗位 JD 详情**：追加新岗位的折叠卡片（包含 JD + 适配分析 + 薪资/地点/匹配度）

### 3. 日程安排甘特图（可选）
- **文件**：`02_速查手册/docs/dashboard.md`
- **操作**：如果投递进展影响日程安排（如收到面试邀请需要调整复习计划），同步更新甘特图

## JD 折叠卡片格式规范

```markdown
??? success "🏢 公司名 — 岗位名 (已投递 YYYY-MM-DD)"

    **岗位职责：**

    1. ...
    2. ...

    **职位要求：**

    1. ...
    2. ...

    ---

    **薪资范围：** XXk · XX薪 | **地点：** 城市-区 | **匹配度：⭐⭐⭐⭐ XX%**
```

### 状态对应的折叠卡片样式
- `✅ 已投递` → `??? success`（绿色）
- `💬 已沟通` → `??? warning`（橙色）
- `📝 待投递` → `??? note`（蓝色）
- `📞 约面` → `??? tip`（青色）
- `❌ 已拒` → `??? danger`（红色）

## 注意事项
1. **数据一致性**：`Resume_Tracker.md` 和 `17_interview_prep.md` 中的投递记录必须保持一致
2. **封板规则**：已投递的简历不可编辑（详见 generate_resume workflow）
3. **JD 来源**：从用户提供的截图中提取 JD 文本，如截图不清晰则标注"JD 待补充"
