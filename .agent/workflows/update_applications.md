# 投递进度更新工作流

## 触发条件
用户提到"更新投递进度"、"帮我记录投递"等关键词。

## 需要同步更新的 3 个位置

### 1. 投递追踪表 (数据源)
- **文件**：`02_速查手册/docs/Resume_Tracker.md`
- **动作**：表格末尾追加新行，更新封板清单。
- **状态规则**：`✅ 已投递`, `💬 已沟通`, `📝 待投递`, `❌ 已拒`, `📞 约面`。

### 2. 面试准备页 (展示层)
- **文件**：`02_速查手册/docs/17_interview_prep.md`
- **更新内容**：
  1. **5.1 投递总览甘特图**：追加新公司。
  2. **5.2 投递清单**：同步 Resume_Tracker 数据。
  3. **5.3 岗位 JD 详情**：追加新岗位的折叠卡片。

### 3. 可视化看板 (可选)
- **文件**：`02_速查手册/docs/dashboard.html`

## JD 卡片规范 (隐私友好)
```markdown
??? success "[公司名/行业] — [岗位名] (已投递 YYYY-MM-DD)"
    **岗位要求：** ...
    **匹配分析：** ⭐⭐⭐⭐ XX%
    **薪资：** XXk · XX薪 | **地点：** [城市]
```

## 注意事项
1. **数据一致性**：根目录两个核心文件必须实时对齐。
2. **隐私屏蔽**：在生成公开展示的卡片时，可对公司具体名称进行脱敏处理。
