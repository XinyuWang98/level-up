# 每日学习进度总结工作流

## 1. 核心数据源
- **搜索分析**：从本地分析服务或 `02_速查手册/data/search_log.json` 读取。
- **Gap Analysis**：读取 `03_职业规划/Gap_Analysis.md` (或 Example 文件)。
- **对话上下文**：读取 `<appDataDir>/brain/` 下今日日期的对话记录。

## 2. 执行步骤
1. **获取搜索分析数据**：提取高频函数与薄弱模块。
2. **扫描对话记录**：提取今日练习、提问及遇到的错误。
3. **更新 Gap Analysis**：在 `## 📝 历史进度追踪` 部分追加今日条目。
4. **生成报告**：路径为 `02_速查手册/reports/daily_YYYY-MM-DD.md`。

## 3. 报告规范 (隐私保护)
- 严禁在报告中提及真实姓名或投递的具体公司。
- 重点评估“资深数分”能力的达成度（如：实验评估的严谨性、代码的生产环境意识）。
- **语言要求**：严格中文（Rule 0），保留必要技术缩写。

## 4. 通知用户
使用 `notify_user` 提交报告路径。
