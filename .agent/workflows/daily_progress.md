# 每日学习进度总结工作流 (Daily Learning Progress)

当用户说"总结一下我今天的学习进度"时，自动执行以下步骤，生成一份完整的学习进度报告。

## 1. 获取搜索分析数据

从本地分析服务获取今日的搜索数据与 AI 建议：

// turbo
```bash
curl -s http://127.0.0.1:5111/api/analyze?period=day 2>/dev/null || echo '{"error": "分析服务未启动，将从日志文件直接读取"}'
```

如果服务未启动，直接读取日志文件作为备选：

// turbo
```bash
cat "./02_速查手册/data/search_log.json" 2>/dev/null || echo "[]"
```

从返回数据中提取：
- **高频搜索函数**：`func_docs` 中的函数名和搜索次数
- **薄弱模块**：`weak_modules` 中的模块名

## 2. 扫描今日对话上下文

回顾今日（当前日期）与用户的所有对话记录，从对话中提取：
- 用户**实际练习过**的函数/概念（如 groupby、merge、fillna 等）
- 用户**提问过**的知识点
- 用户**完成的练习**或 notebook
- 用户**遇到的错误**及是否已解决

方法：检查 `<appDataDir>/brain/` 下今日日期的对话目录，读取 overview.txt 和任务日志。

## 3. 更新 Gap Analysis

读取并更新 Gap Analysis 文档：

// turbo
```bash
cat "./05_职业规划/Gap_Analysis_Example.md"
```

在 `## 📝 历史进度追踪 (Progress Log)` 部分追加今日条目：

```markdown
### 📅 YYYY-MM-DD (Day X)
*   **[+ New]** 今日新掌握的技能/概念...
*   **[- Gap]** 今日暴露的薄弱点...
*   **[📊 Search]** 今日高频搜索函数（来自分析服务）
```

使用 `replace_file_content` 工具在 Progress Log 部分追加条目。

## 4. 生成每日学习报告

在以下路径创建独立的 Markdown 报告文件：

**路径**: `./02_速查手册/reports/daily_YYYY-MM-DD.md`

报告模板：

```markdown
# 📊 每日学习进度报告 - YYYY-MM-DD

> ⏰ 生成时间：HH:MM | 📅 学习天数：Day X

## 🔍 今日搜索画像

| 函数/概念 | 搜索次数 | 官方文档 | 掌握程度   |
| --------- | -------- | -------- | ---------- |
| groupby   | 5        | [链接]() | ⚠️ 反复查询 |
| merge     | 2        | [链接]() | ✅ 已熟练   |

## 💬 今日对话总结

### 练习完成
- [x] 完成 Module C 过滤练习
- [ ] ...

### 涉及函数/概念
- `groupby` - 在 XX 练习中使用
- `boolean indexing` - 在过滤练习中复习

### 遇到的问题
- 问题描述 → ✅ 已解决 / ⚠️ 待跟进

## 🎯 薄弱模块定位

根据搜索频次 + 对话中的表现，以下模块需要加强：
1. 🐼 Pandas 高级操作（groupby + agg 组合）
2. ...

## 💡 Agent 建议

### 是否需要加练？

基于以下因素综合判断：
- **当前时间**：如果是晚上 9 点之后，建议明天再练
- **今日练习量**：如果已完成 ≥ 2 个模块，建议休息
- **薄弱点紧迫性**：如果涉及面试高频考点，建议尽快安排
- **连续学习天数**：如果连续 ≥ 5 天高强度，建议休息日

### 推荐行动
1. 📝 [具体建议，如"明天优先练习 groupby + agg 组合"]
2. 📖 [推荐阅读的文档或 cheatsheet 页面]
3. 💪 [如果建议加练，给出具体的 Prompt]
```

## 5. 通知用户

使用 `notify_user` 工具：
- 设置 `PathsToReview` 为生成的报告文件路径
- 在 `Message` 中简要总结今日亮点和建议
- 如果建议加练，附上可直接复制的练习 Prompt

## 执行注意事项

1. **时间感知**：使用系统当前时间判断日期和时段
2. **无数据处理**：如果今日没有搜索记录或对话，报告应如实反映，并建议用户开始学习
3. **加练判断逻辑**：
   - 当前时间 < 21:00 且 薄弱模块 ≥ 2 → 建议加练 30 分钟
   - 当前时间 ≥ 21:00 → 建议明天优先处理
   - 今日已练 ≥ 3 小时 → 建议休息
4. **语气**：专业但鼓励，像 Senior 带 Junior 做复盘
