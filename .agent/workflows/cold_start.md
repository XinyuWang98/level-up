# /cold_start 初始化工作流

当新用户第一次 Fork 本仓库并唤醒 Agent 时，应执行此工作流。

## 1. 身份对齐 (Identity Sync)
- **触发提问**：Agent 询问用户当前的职业背景（简历）及目标岗位（JD）。
- **动作**：读取用户提供的简历内容，识别核心技术栈与行业背景。

## 2. 叙事映射初始化 (Narrative Mapping)
- **分析转型路径**：根据“当前背景”与“目标岗位”的差异，识别需要升维或包装的术语。
- **生成文件**：在 `03_职业规划/` 下创建或更新 `Narrative_Mapping.md`。
- **模板填充**：
    - 填充专业化术语对照表。
    - 定义 3 个核心业务场景的升维路径。

## 3. 导师角色确认 (Mentor Calibration)
- **更新规则**：根据用户的目标（如 P7 数分、全栈开发），引导用户确认 `.agent/rules/learning.md` 中的 `User` 和 `Agent` 设定。
- **调整风格**：确认为“挑战模式 (Challenge Mode)”还是“教学模式”。

## 4. 首次能力体检 (First Benchmark)
- **执行指令**：自动或手动引导用户执行一次 `/generate_gap_analysis`。
- **输出成果**：生成首份 Gap Analysis 报告副本。

## 5. 知识库启动
- **动作**：告知用户如何运行 `02_速查手册/start.sh` 来预览本地知识库。
