## 触发时机

- 某个 cheatsheet 新增了 **3 个以上知识点** 后
- 用户主动请求优化某个 cheatsheet
- `/update_cheatsheet` 执行完毕后的可选后续步骤

## 前置条件

- 目标文件已存在于 `./02_速查手册/docs/` 下
- MkDocs 服务正在运行 (`mkdocs serve`)

## 审查清单 (Checklist)

### A. 内容完整性审查

1. **是否有方法/概念对比总表？**
    - 如果 cheatsheet 涉及 **3 种以上可对比的方法/工具/概念**，必须有一张横向对比表 (Markdown Table)。
    - 表头建议：方法名 | 适用场景 | 核心假设/前提 | 优点 | 缺点/局限 | 关键参数。
    - 放在 SOP / 概念总览章节之后，具体方法章节之前。

2. **是否有决策树/流程图？**
    - 如果 cheatsheet 涉及 **方法选型** (何时用 A vs B)，必须有 Mermaid 决策流程图。
    - 格式：```` ```mermaid graph TD ... ``` ````，使用颜色区分终端节点。
    - 放在对比总表之前或并列展示。

3. **是否有面试 Q&A 章节？**
    - 每个 cheatsheet 至少 5 个高频面试问题。
    - 使用 `??? question "Q1: 问题"` 折叠格式。
    - 问题覆盖：方法选型、假设违反、局限性、向非技术人员解释、实战踩坑。

4. **是否有业务汇报话术模板？**
    - 至少 1 个示例，使用 `>` blockquote 格式。
    - 包含具体数字占位符 (e.g. **X 元**, **X%**)。

### B. 展示形式审查

1. **代码模板是否使用折叠块？**
    - 超过 10 行的代码块必须包裹在 `??? example "标题 (点击展开)"` 中。
    - 折叠块内的代码和列表需要 **4 空格缩进**。
    - 短代码 (≤10 行, 如单个函数调用) 可以保持展开。

2. **列表前是否有空行？**
    - 段落文字后紧跟 `*` 或 `1.` 列表时，**必须** 插入空行。
    - 否则 MkDocs 会将列表吞成行内文本。

3. **箭头符号是否正确？**
    - 代码块外部：使用 `→` (Unicode 箭头)，**不要** 使用 `->`。
    - 代码块内部：保持 `->` 不变。

4. **LaTeX 公式是否能渲染？**
    - 块级公式：使用 `$$..$$` 包裹。
    - 行内公式：使用 `$...$` 包裹 (MathJax 模式)。
    - 如果 MkDocs 使用 arithmatex 扩展，行内格式可能需要 `\(...\)`。
    - **验证方法**: 在浏览器中打开页面，检查公式是否渲染为数学符号。

5. **Admonition (提示框) 使用是否规范？**
    - `!!! tip`: 面试速记、实战建议
    - `!!! warning`: 常见坑、数据陷阱
    - `!!! note`: 补充说明
    - `??? question`: 面试 Q&A (折叠)
    - `??? example`: 代码模板 (折叠)
    - `??? note`: 详细说明 (折叠)
    - 内部内容需要 **4 空格缩进**。
    - **admonition 内多段文字必须用空行分隔**：连续的缩进行会被 MkDocs 合并为一个段落。如果希望换行显示（如"❌ 陷阱"和"✅ 正确答法"分两段），中间必须插入一个仅含 4 空格缩进的空行。

6. **章节编号是否连续？**
    - 检查 `##` 级别的编号是否从小到大依次递增，无跳跃或重复。

### C. Mermaid 图规范

- 节点标签含特殊字符时用双引号包裹：`A["Label (Extra)"]`
- **双主题可见性 (铁律)**：所有节点必须在日间主题和夜间主题下都清晰可见。
    - 使用 **中等亮度** 的填充色 (避免过深如 `#1b4332` 或过浅如 `#95d5b2`)
    - 文字统一用 **白色** (`color:#fff`)
    - 必须添加 **描边** (`stroke:#深色,stroke-width:2px`) 以确保浅色背景下节点边界清晰
    - 推荐色板 (已验证双主题兼容)：
        - 绿色系: `fill:#1a7f37` / `fill:#2da44e`
        - 蓝色系: `fill:#0969da`
        - 紫色系: `fill:#8250df`
        - 黄色系: `fill:#bf8700`
        - 灰色系: `fill:#57606a`
        - 红色 (失败): `fill:#cf222e`
- 不要在标签中使用 HTML 标签

## 执行步骤

1. 读取目标 cheatsheet，统计现有章节结构。
2. 逐项对照上述 Checklist (A + B + C)，记录缺失项。
3. 按优先级修复：
    - 🔴 P0: 对比总表、决策树 (若缺失) → 直接插入
    - 🟡 P1: 代码折叠、Q&A 补充 → 批量修改
    - 🟢 P2: 格式微调 (空行、箭头、编号) → 快速修复
// turbo
4. 在浏览器中验证修复效果 (`http://127.0.0.1:8000/{filename}.html`)。
5. 通知用户完成的优化项和验证截图。
