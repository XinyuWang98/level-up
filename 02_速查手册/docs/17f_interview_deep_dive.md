## 二、分层重叠实验框架核心设计

??? question "🎯 底层分流逻辑怎么设计的？什么分桶方法？怎么保证正交性？时区绑定问题怎么解决？"

    ### 分流架构：分层 + 域隔离双层设计

    #### 分桶方法
    - **MurmurHash3 哈希算法**，对用户 UUID 哈希后对 1000 取模
    - 1000个等宽分桶，单分桶流量占比 0.1%
    - 选择原因：哈希均匀性好（分桶用户量差异 ±0.5%）、性能高、无状态
    - UUID 不变则分桶永久固定，避免用户跨组跳转

    #### 分层规则
    - 4个互斥层：智能话术、用户路由、人工排班、兜底功能
    - **同层实验互斥、跨层实验正交**

    #### 正交性保证（三重机制）

    1. **分层盐值隔离**：每个层配置独立盐值，对同一 UUID 用不同层盐值做二次哈希分桶，层间相关系数为 0
    2. **层内互斥强制校验**：同层实验流量分桶有重叠则无法上线
    3. **正交性常态化诊断**：看板每日计算跨层用户重合度，偏差超 1% 触发告警

    #### 时区绑定方案
    - 市场-时区唯一映射表，实验选市场后自动绑定时区
    - 时段配置仅支持市场本地时段，系统自动转 UTC 做分流
    - 指标统计按市场本地时间自然日聚合
    - 跨时区市场强制分市场设置时段，禁止统一配置

??? question "🎯 有没有出现过分流不均的情况？你是怎么解决的？"

    上线初期遇到 **2次** 典型分流不均问题：

    #### 问题1：小流量实验分桶属性不均
    - **现象**：1% 流量实验 SRM 检验 p<0.05，实验组与控制组属性分布不均
    - **原因**：小流量下随机波动大，单分桶用户量少
    - **解决方案**：
        - 新增最小流量阈值（单实验 ≥5%）
        - 小流量实验强制分层抽样分桶
        - 上线 SRM 实时校验（1h/6h/24h 三次校验）
        - 新增事后 PSM 匹配校正功能

    #### 问题2：新用户分流重复入组
    - **现象**：新用户注册后立即进线，UUID 未同步导致被分到多个实验组
    - **解决方案**：
        - 新用户首次进线优先用设备 ID 分桶，UUID 同步后平滑切换
        - 新增去重逻辑，确保单用户唯一入组
        - 重复入组率超 0.1% 立即告警

    **优化效果**：分流不均发生率从 8% 降至 <1%，SRM 通过率 99%+

## 三、统计原理硬核拷问

??? question "🎯 推导一阶 Delta Method 对比率型指标的方差公式，什么场景下会失效？"

    ### 方差公式推导

    比率型指标 $R = \frac{X}{Y}$（X=流失会话数，Y=总会话数），样本估计量 $\hat{R} = \frac{\bar{X}}{\bar{Y}}$

    对 $g(\mu_x, \mu_y) = \frac{\mu_x}{\mu_y}$ 做一阶泰勒展开：

    $$g(\bar{X}, \bar{Y}) \approx g(\mu_x, \mu_y) + \frac{1}{\mu_y}(\bar{X}-\mu_x) - \frac{\mu_x}{\mu_y^2}(\bar{Y}-\mu_y)$$

    取方差得：

    $$\boxed{Var(\hat{R}) \approx \frac{1}{n \cdot \mu_y^2} \left[ Var(X) + R^2 Var(Y) - 2R \cdot Cov(X,Y) \right]}$$

    ### 失效场景

    | 场景                       | 原因                                         |
    | :------------------------- | :------------------------------------------- |
    | **分母期望接近0**          | $\mu_y \to 0$ 时偏导数爆炸，一阶近似误差极大 |
    | **样本量极小（n<30）**     | 大样本渐近理论不成立                         |
    | **极度偏态分布**           | 用户进线次数高度右偏，均值方差估计偏差大     |
    | **比率极端值（接近0或1）** | 函数非线性极强，一阶展开无法有效近似         |

    ### 实际遇到的失效案例
    中东小语种市场实验：样本量 <1000，进线分布极度偏态，核心流失率仅 2%

    **短期应急**：改用 Bootstrap 自助法 + 置换检验（10000次重抽样）

    **长期优化**：平台新增场景自适应检验切换逻辑：
    - 样本量 <5000 → 自动切换 Bootstrap
    - 变异系数 >2 → 自动切换 Bootstrap
    - 比率 R<5% 或 >95% → 自动切换 Bootstrap

??? question "🎯 CUPED 协变量选的什么？为什么选它？30% 方差缩减怎么来的？有没有出现过负缩减？"

    ### 协变量选择
    **实验前7天用户的个体核心流失率**（流失会话数/总会话数）

    新用户采用同市场、同时段、同会员等级用户的平均核心流失率

    ### 选择理由

    | 理由           | 说明                              |
    | :------------- | :-------------------------------- |
    | **相关性极高** | ρ = 0.6-0.7，理论方差缩减 36%-49% |
    | **外生性满足** | 实验前数据，不受实验干预影响      |
    | **场景适配**   | 导购/推荐场景用户流失行为惯性极强    |

    有效阈值：**ρ ≥ 0.3**（缩减 ≥9%）才有业务价值，ρ ≥ 0.5 效果显著

    ### 30% 的来源
    平台上线后 **30+ 正式实验**的方差缩减均值：
    - 智能话术类：35%-40%
    - 排班策略类：20%-25%
    - 全量均值：**30%**

    ### 负方差缩减（2次）

    | 次序  | 原因                                         | 解决方案                               |
    | :---: | :------------------------------------------- | :------------------------------------- |
    | 第1次 | 协变量选了实验前**1天**，恰逢大促，ρ = -0.12 | 固定协变量窗口为实验前7天完整自然日    |
    | 第2次 | 新用户实验的协变量与实验窗口重叠，违反外生性 | 新用户禁用注册后行为数据，仅用固有属性 |

    平台新增自动校验：ρ < 0.1 或为负时，自动告警并切回原始方差计算

## 四、因果推断落地案例（DID）

??? question "🎯 讲一个你用 DID/PSM/DoWhy 落地的完整项目，包括背景、模型设计、识别假设、成果"

    ### 项目背景
    2023年上半年，欧美市场核心流失率居高不下。策略团队计划上线**意图匹配智能安抚话术**，但 A/B 实验平台仍在开发中，仅允许在英国市场试点，无法做随机对照实验，因此采用 **DID 双重差分法**评估策略效果。

    ### 模型设计
    - **处理组**：英国市场（2023.3.1 上线意图匹配话术）
    - **控制组**：德国市场（用户结构、进线量、核心流失率趋势高度相似，全程未上线）
    - **时间窗口**：干预前8周 / 干预后8周

    $$Y_{it} = \alpha + \beta \cdot Treat_i \cdot Post_t + \gamma \cdot X_{it} + \mu_i + \lambda_t + \varepsilon_{it}$$

    - $\beta$ = 核心处理效应
    - $X_{it}$ = 控制变量（周订单量、进线量、新用户占比、促销虚拟变量）
    - $\mu_i$ = 市场个体固定效应，$\lambda_t$ = 时间固定效应

    ### 识别假设检验

    | 假设           | 验证方法                         | 结果   |
    | :------------- | :------------------------------- | :----- |
    | **平行趋势**   | 事件研究法，干预前各周系数 p>0.1 | ✅ 满足 |
    | **SUTVA**      | 英德市场用户无重叠，无策略溢出   | ✅ 满足 |
    | **共同冲击**   | 干预期间两市场其他策略完全同步   | ✅ 满足 |
    | **无预期效应** | 话术为后端改动，用户无法提前知晓 | ✅ 满足 |

    ### 落地成果
    - $\beta = -0.032$（p<0.01）：英国市场核心流失率显著下降 **3.2pp**
    - 推广至全欧5个市场，整体下降 2.8pp，每月节省 1200+ 人力工时

    ### 与 A/B 实验的对比验证
    2023年8月平台上线后，法国市场做了同策略的 A/B 实验：
    - A/B 结果：下降 **3.1pp**（p<0.01）
    - DID 结果：**3.2pp**
    - 偏差仅 0.1pp，远低于 5% 可接受范围 ✅

## 五、业务成果归因验证

??? question "🎯 用户行为轨迹 体系推动核心流失率从37%降到28%,你怎么证明9pp的降幅中诊断体系的独立贡献度？拆分6个痛点各贡献多少？"

    ### 项目周期与并行项目
    - **完整周期**：2023.10 启动 → 2024.6 全量落地，共 8 个月

        - 2024.1 个性化推荐/核心系统意图识别模型迭代

    ### 三层归因方法

    #### 层1：断点回归验证即时效应
    以 2023.12.15 体系全量上线为断点，控制订单量、进线量、季节性趋势，发现上线2周内核心流失率下降 **1.2pp**（此时段其他项目均未上线 → 100% 归因于诊断体系）

    - 处理组：4个深度使用体系的市场（使用率 90%+）

    #### 层3：事件研究法逐个验证痛点贡献
    每个痛点修复有明确上线时间，验证修复后对应场景核心流失率下降 + 其他场景无显著变化

## 六、高级专家 定级能力边界

??? question "🎯 如果让你把导购/推荐实验体系复制到用户增长业务线，你会做哪些核心调整？两个场景本质区别是什么？"

    ### 五大核心调整

    1. **实验分层重构**：从业务模块分层 → 用户旅程正交分层（拉新/激活/转化/复购/留存），新增全局互斥层 + 触点域隔离
    2. **分流引擎重构**：从静态用户粒度 → 多粒度自适应分流（设备ID/用户ID/订单ID/曝光ID），支持实时行为触发动态分流
    3. **统计引擎重构**：新增零膨胀模型 + Bootstrap（付费率等）、贝叶斯序贯检验（7日留存/30日LTV）、FDR 多重检验校正
    4. **护栏体系重构**：单业务线 KPI → 三级全公司级护栏（公司级/增长线/模块级），高风险实验互斥锁
    5. **平台能力扩展**：单场景 → 全链路，对接广告/CRM/优惠券/推荐系统，新增 30/90 天长期效果追踪

    ### 本质区别

    | 维度     | 导购/推荐实验               | 增长实验                 |

    | 分流引擎 | 静态单粒度             | 动态多粒度               |
    | 统计引擎 | 短周期正态指标         | 长周期非正态/零膨胀      |
    | 风险管控 | 单业务线隔离           | 全公司级多级管控         |
