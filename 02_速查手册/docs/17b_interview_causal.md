## 💡 第二步：核心概念卡片

!!! info "Correlation ≠ Causation"
    冰淇淋销量 ↑ 和溺水人数 ↑ 高度相关，但因果链是：**温度（混淆变量）** 同时驱动了两者。
    因果推断的核心就是**识别并控制**这些混淆变量。

!!! info "PSM vs IPW：微观个体级的偏差修复"
    **PSM**："找替身"——在对照组里严格 1:1 为实验组用户匹配一个特征相似的双胞胎。
    **IPW**："加放大器"——保留所有样本，用权重 $w = \frac{p}{1-p}$ 把少数派人群的影响力放大。
    **何时选谁**：对照组多 → PSM（丰水池找替身）；对照组少 → IPW（不浪费珍贵样本）。

!!! info "DID：时间×分组的双重差分"
    治疗效果 = (实验组后-实验组前) - (对照组后-对照组前)。
    **硬性前提**：平行趋势假设——如果没有干预，两组的时间趋势应该平行。
    **验证方法**：画实验前时间序列图 + 安慰剂检验（假 DID 系数应≈0）。

!!! info "SCM vs PSM：宏观 vs 微观"
    **SCM** 是用来对付"无法做用户级随机分流的宏观聚合实体"（如一个城市）。
    **PSM** 是用来对付"用户级的选择性偏差"（如 A/B 测试分流坏了）。
    **面试陷阱**：在用户级 A/B 实验中提 SCM 会减分！

!!! info "稳健性检验：PSM 和 IPW 都需要的 Balance Check"
    无论是扔数据找替身（PSM）还是加权拉扯（IPW），做完之后都必须验证：
    1. **SMD < 0.1**：所有核心特征的标准化均值差异被压到 0.1 以内
    2. **Placebo Test**：把权重/匹配套到历史数据上，ATE 应 ≈ 0

## ❓ 第三步：面试巩固题

### 基础题

??? note "Q1: PSM 的原理和步骤？局限性？"
    **核心答法**：Logistic 回归估计 PS → 匹配（最近邻/卡尺）→ Balance Check (SMD < 0.1) → 估计 ATT
    **局限**：无法控制未观测混淆变量 (CIA 硬伤)

??? note "Q2: DID 适用条件？时间趋势不一致怎么办？"
    **核心答法**：前提是平行趋势假设 → 可视化检验 → 不满足时可加时间趋势交互项或转用 SCM

### 进阶题（来源：模拟面试）

??? warning "Q3: 转人工率下降 9pp，如何归因到你的项目？ ⭐必练"
    **来源**：Gemini 模拟面试
    → [查看降维打击回答：难题 1](17_interview_prep.md#hard-flashcards)
    **骨架记忆**：Phased Rollout → PSM 匹配渠道特征 → DID 剔除时间趋势 → SRM 验证

??? warning "Q4: DID 平行趋势假设怎么验证？如果不平行呢？ ⭐杀手锏题"
    **来源**：Gemini 模拟面试 Q4
    **答法**：
    1. 画实验前两组时间序列图，观察趋势是否平行
    2. 安慰剂检验：在实验前做一次"假 DID"，系数应接近 0
    3. 如果不平行：改用 PSM-DID（先匹配再差分）或 SCM（合成控制法）

??? warning "Q5: SRM 后用 IPW 挽救，业务说你在造假，怎么办？ 😈硬核"
    → [查看降维打击回答：难题 6](17_interview_prep.md#hard-flashcards)
    **骨架记忆**：SMD Balance Check（加权后特征对齐）+ Placebo Test（把权重套到历史数据，ATE≈0 证明权重合法）

### 🎤 豆包模拟面试高频题 (2026.02.27 — 因果推断专场)

> 以下题目来自豆包基于简历生成的因果推断专场考题，按业务深度递进排列。

#### 板块一：A/B 实验中的因果推断（与简历 Story 3 强绑定）

??? warning "Q6: 实验指标正向但核心指标负向，如何用因果推断证明策略导致的？ ⭐必练"
    **骨架答法**：
    1. 先排除混淆：检查实验期间是否有外部活动/版本更新（时间混淆）
    2. 用 **中介效应分析 (Mediation Analysis)**：拆解 策略→决策指标→核心指标 的因果链路
    3. 如果核心指标负向与策略有因果关系 → 一票否决，护栏红线不可逾越

??? note "Q7: A/B 实验里的混淆变量是什么？举一个业务中最常见的例子"
    **核心答法**：混淆变量 = 同时影响分组和结果的第三方因素。
    **业务例子**：用户活跃度。高活跃用户更容易被触发实验（曝光偏差），同时高活跃用户本身转化率就高 → 如果不控制活跃度，会高估策略效果。
    **解决**：分层随机化 / CUPED 用实验前活跃度做协变量

??? note "Q8: SRM 通过了，为什么因果识别还可能不干净？"
    **核心答法**：SRM 只保证"人分对了"，不保证"人的行为没被污染"。
    三类隐患：1) **新奇效应 (Novelty Effect)**：用户因为新功能而暂时行为异常；2) **网络效应 (Spillover)**：实验组用户影响了对照组用户；3) **存活偏差 (Survivorship Bias)**：只看了完成实验的用户，忽略了中途流失的

??? warning "Q9: 什么是处理效应异质性 (HTE)？不同用户群效果相反怎么办？ ⭐加分题"
    **核心答法**：HTE = 同一策略对不同人群效果不同甚至相反（辛普森悖论的因果推断版本）。
    **业务例子**：新客服话术对高价值用户降低了转人工率，但对低价值用户反而提高了（因为话术太复杂）。
    **解法**：CATE 估计（条件平均处理效应）→ 按用户分层做子群 A/B → 如果有能力，用因果森林 (Causal Forest) 精准定位

??? warning "Q10: 实验溢出效应 (Spillover) 怎么修正？"
    **核心答法**："溢出"= 实验组的行为间接影响了对照组（如社交推荐、共享库存）。
    **排查**：检查对照组在实验期间的指标是否异常偏移（与 AA 基线对比）。
    **修正**：1) 集群随机化（按地区/城市分流，物理隔离）；2) 排除社交链路上的用户；3) 用 IPW 对溢出程度做加权校正

#### 板块二：不能做 A/B 时的观测数据因果方法（高级专家 核心区分点）

??? warning "Q11: 业务里哪些场景绝对不能做 A/B？用什么因果方法？ ⭐必练"
    **核心答法**（三类场景）：
    1. **伦理约束**：不能把部分用户分到"差体验组"（如医疗/安全）→ 用 **PSM/IPW**
    2. **技术不可分流**：全量上线的政策变更（如运费规则）→ 用 **DID**
    3. **样本太少**：只有一个城市试点 → 用 **SCM (合成控制法)**

??? warning "Q12: PSM 的三个核心假设是什么？什么时候不能用？ ⭐必练"
    **三大假设**：
    1. **CIA (条件独立假设)**：控制了可观测变量后，分组与结果独立 → 最大软肋：无法处理未观测混淆
    2. **Common Support (共同支撑)**：实验组和对照组的倾向得分分布必须有足够重叠
    3. **SUTVA (稳定单元处理值假设)**：一个个体的结果不受其他个体是否被处理影响（无溢出效应）
    **不能用的场景**：怀疑有强未观测混淆变量（如用户"动机"这种无法量化的因素）→ 应转用 IV 或 RDD

??? note "Q13: DID 适合你业务里的什么场景？平行趋势怎么验证？"
    **业务适配**：适合"某个时间点全量上线了新策略"的前后对比，如"新版知识库 3/1 上线，对比上线前后转人工率变化"。
    **平行趋势验证**：1) 画图看实验前两组趋势是否平行；2) Event Study 动态效应图；3) Placebo Test (假时间点 DID 系数≈0)
    **不满足时**：PSM-DID（先用 PSM 匹配使两组更可比，再做 DID）或 SCM

??? note "Q14: RDD 在你的业务里能用来做什么？"
    **核心答法**：RDD 需要一个"明确的分界线"，分界线两侧的处理逻辑不同。
    **业务例子**："当会话轮数 ≥ 5 轮时自动触发转人工" → 围绕 5 轮这个断点，对比 4 轮 vs 6 轮用户的后续满意度，就是天然的 RDD。
    **注意**：需要 Manipulation Test（排除用户故意操控断点变量的可能）

??? note "Q15: IV 工具变量你用过吗？业务里能找到什么合法的 IV？"
    **核心答法**：IV 需要满足 1) 与处理变量相关 2) 只通过处理变量影响结果（排他性约束）。
    **业务例子**：要评估"加入 VIP 客服通道"对留存的影响，可以用"推送 VIP 邀请弹窗的时间随机性"作为工具变量（弹窗时刻影响是否加入 VIP，但弹窗本身不直接影响留存）。
    **坦诚防守**："在实际业务中寻找完美的 IV 是极其困难的，我更多依赖 PSM/DID，但理解 IV 的思想有助于我评估因果结论的稳健性。"

#### 板块三：业务实战归因（落地场景题）

??? warning "Q16: 大盘留存突然跌了，怎么用因果推断而非相关性找根因？ ⭐必练"
    **骨架答法**：
    1. **维度下钻**：按用户群/渠道/时段拆解，定位是全面下跌还是局部下跌
    2. **排除混淆**：检查同期是否有外部事件（大促结束、竞对活动、版本 Bug）
    3. **因果验证**：对疑似根因做 **DID / PSM 反事实推演**（如：怀疑是新版本导致的 → 对比升级用户 vs 未升级用户的留存差异，控制用户特征）
    4. **Granger 因果检验**：用时序方法验证"根因变量"是否在时间上领先于"留存下跌"

??? warning "Q17: 转人工率上升，怎么区分是策略/用户结构/外部活动导致的？"
    **骨架答法**（三刀切分法）：
    1. **控制用户结构变量**：用 PSM/IPW 匹配新老用户比例、渠道来源 → 排除用户结构变化
    2. **控制时间趋势**：用 DID 对比有无策略变更的时间段 → 排除自然趋势
    3. **控制外部活动**：检查大促/系统故障日历 → 如果与外部事件重合，需剔除该时段数据或加入交互项

??? note "Q18: 怎么判断指标上涨是真因果还是同期混杂带来的假因果？"
    **核心答法**：做 **安慰剂检验 (Placebo Test)**。
    把你的"因果模型"套到一个你**确信没有因果效应**的场景上（比如换一个无关指标、换一个未受策略影响的用户群）。如果模型依然给出"显著效果" → 说明你的模型在捡垃圾相关性，原始结论不可信。

??? warning "Q19: 什么是中介效应？你的策略通过什么中间行为影响核心指标？ ⭐加分题"
    **核心答法**：中介效应 = 策略不是直接作用于核心指标，而是通过一个"中间变量"间接影响。
    **业务例子**：知识库优化 → 提升了 Bot 首轮匹配准确率（中介变量）→ 降低了转人工率（核心指标）。
    **验证方法**：Baron & Kenny 三步法 或 Sobel Test → 拆解直接效应 vs 间接效应的占比

#### 板块四：高阶因果 + 业务决策

??? warning "Q20: 因果推断和 A/B 实验是什么关系？为什么有了实验还要因果？ ⭐必练"
    **核心答法**：
    > "A/B 实验是因果推断的**黄金标准**，但不是万能药。
    > 1) 有些场景**不能做 A/B**（伦理/技术/成本）→ 需要 PSM/DID/IV 等观测数据方法
    > 2) A/B 只告诉你**整体 ATE**，不告诉你**谁受益谁受损** → 需要 HTE/因果森林
    > 3) A/B 结果可能被**溢出效应**污染 → 需要因果图/结构方程模型辅助诊断
    > 因果推断是 A/B 实验的**上位理论框架**，A/B 是其中最干净的一种特例。"

??? note "Q21: 小流量实验不显著，但怀疑有因果效应，怎么提升检出力？"
    **骨架答法**（三板斧）：
    1. **CUPED 降噪**：用实验前的协变量压缩方差 → Z-Score 放大 → 更容易击穿 P 值门槛
    2. **分层分析**：可能整体不显著但某子群显著（HTE），按高价值/低价值用户分层看
    3. **延长实验**：如果 Power 不够就是样本量不足 → 把实验周期从 3 天延长到 7 天

??? note "Q22: 如何给业务讲清楚：相关性≠因果性？举一个你踩过的坑"
    **业务例子**：曾经发现"使用帮助中心的用户，转人工率反而更高"。
    业务第一反应是"帮助中心没用，砍掉！"。
    但用因果图分析发现：**问题复杂度**是混淆变量——问题越复杂的用户，越倾向于先看帮助中心（但最终还是要转人工）。
    帮助中心降低了简单问题的转人工，但对复杂问题无能为力 。用 PSM 控制问题复杂度后，帮助中心实际上使转人工率下降了 3pp。
