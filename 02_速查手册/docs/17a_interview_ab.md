## 💡 第二步：核心概念卡片（从视频衍生的关键理解）

> 看完视频后，确认自己能理解以下每一条。如果某条看不懂，回去重看对应视频。

!!! info "P-value 本质与极易踩坑的面试陷阱"
    P 值 = **假设策略没用（H₀为真）的前提下**，观察到当前或更极端结果的概率。
    P=0.03 意味着：如果策略真的没效果，单凭随机噪音产生这么大差别的概率只有 3%。

    ❌ **面试官常挖的坑**：“实验算出来 P=0.03，是不是说明你的策略有 97% 的概率是有效的？”

    ✅ **防守话术**：“不是的。P 值只能说明由于概率太小（小于 0.05），我不相信这是随机噪音，所以我拒绝原假设。但这绝不代表策略有效性的概率，因为判断策略到底多大程度上有效，必须看 ATE（效应量）和置信区间。”

!!! info "样本量计算 (MDE)"
    4 个输入决定你需要多少人：**α**（假阳性容忍度）、**1-β**（检出力）、**MDE**（最小可检测效应）、**σ²**（基线方差）。
    **业务语言**："如果这个策略至少能让转化率提升 2%，我需要跑 14 天 × 每天 1 万用户才能检测到它。"

!!! info "SRM (Sample Ratio Mismatch)"
    分流坏了，一切白搭。卡方检验 p < 0.001 就报警。
    **常见原因**：Bot 流量、触发条件 Bug、Hash 碰撞、重定向延迟。

!!! info "CUPED 本质"
    用实验前的历史指标（协变量 X）修正实验期指标（Y），**剥离用户固有差异，只测策略纯增量**。
    $Y_{cuped} = Y - \theta \cdot (X - \bar{X})$，其中 $\theta = \frac{Cov(X,Y)}{Var(X)}$。
    **五步稳健性检验**：均值不变 → 方差缩减 → Pre-experiment → 全局 θ → Uplift 不变。

!!! info "Delta Method 本质"
    比率指标（转化率 = 转化数/UV）的分子分母都会随机波动。
    不能简单用二项分布算方差，必须用一阶泰勒展开同时考虑协方差。

!!! info "Sequential Testing (DTD vs LTD)"
    **DTD**（每日独立切片）：数据不重叠，可直接用 α=0.05。
    **LTD**（累积至今）：每次偷看 = 对同批人反复检验 → 假阳性膨胀 → 必须用 **Alpha Spending** 动态调门槛。

!!! tip "🥇 高级专家 黄金答题框架：如何从头到尾评估一个 A/B 实验？"
    **第一步看 SRM（分流健康度）**：分流不均衡，后面一切数据免谈。
    **第二步看 P-value（统计显著性）**：P < 0.05 说明变化不是随机噪音，有底气“拒绝原假设”。但这只是及格线，不代表策略好。
    **第三步看 ATE 和置信区间下界（业务显著性与防守底线）**：评估增量有多大，更重要的是看 **95% 置信区间的下界（Lower Bound）**是否依然高于业务成本线/盈亏平衡点。只有最差情况也能 cover 成本，才决策全量上线。
    > *"概括来说：我们只是站在概率的尺子上给出是不是噪音的论断（P值）；真正决定生死的，是第三步的效应量评估（ATE+CI）。"*

## ❓ 第三步：面试巩固题（检验你是否真正理解了）

> 先看题目尝试口头回答，再点开对照。如果能流畅地用自己的话讲 80%，这道题就算过了。

### 基础题

??? note "Q1: CUPED 方差缩减原理？注意事项？"
    **核心答法**：用实验前的协变量回归调整，降低方差。
    注意：协变量必须在实验前确定，不能用实验期间的数据；θ 必须用全局数据计算。

??? success "✅ Q2: SRM 的可能原因？如何检测和修复？"
    **核心答法**：卡方检验检测 → 常见原因（Bot、触发 Bug、Hash 碰撞）→ 修复（过滤异常流量、修复分桶逻辑、重跑实验）

??? note "Q3: Delta Method 是什么？为什么比率类指标需要它？"
    **核心答法**：比率指标是随机变量的函数 → 方差不能直接用二项分布 → Delta Method 用一阶泰勒展开逼近方差

??? note "Q4: 如何确定实验所需样本量？"
    **核心答法**：4 个输入（α、1-β、MDE、基线方差）→ 公式或在线计算器 → 实际中常需与业务协商 MDE

??? note "Q5: 分层实验如何防止交叉污染？Spillover 怎么处理？"
    → [查看降维打击回答：难题 4](17_interview_prep.md#p7-4)

### 进阶题（来源：模拟面试）

??? warning "Q6: 实验组转化率 +5%，P=0.04，但 SRM 报警。结果可信吗？ 🟢"
    **来源**：Gemini 模拟面试 #2
    **答法**：SRM 意味着随机分流遭到破坏 → P-value 无意义 → 必须先排查 SRM 原因 → 修复后重跑

??? warning "Q7: P=0.08 未显著，业务急着上线怎么办？ ⭐必练"
    → [查看降维打击回答：难题 5](17_interview_prep.md#hard-flashcards)
    **骨架记忆**：排查 SRM/MDE → PSM/IPW 修复偏差 → CUPED 降噪

??? warning "Q8: 实验组比对照组少/多 5%，怎么挽救？ 😈硬核"
    → [查看降维打击回答：难题 6](17_interview_prep.md#hard-flashcards)
    **骨架记忆**：少→PSM（丰水池找替身）| 多→IPW（枯水池加权拉扯）→ SMD + Placebo 验证

??? warning "Q9: CUPED 在大促期间还适用吗？（协变量分布变了）"
    **来源**：DeepSeek 模拟面试 Q3
    **答法**：需检验实验前与实验期间的协变量分布是否稳定 → 如果大促导致用户结构剧变，可引入时间趋势交叉项 → 或者只使用大促期间的历史数据作为协变量

??? warning "Q10: 分析周期 7→3 天，用了什么加速方法？ ⭐必练"
    **骨架记忆**：三层加速 = **Delta Method** 校正 Ratio Metric 方差 + **CUPED** 方差缩减 30%+ + **Sequential Testing** 支持中途决策

### 概念深挖题（来源：视频学习 2026.02.26）

??? warning "Q11: α 和 P-value 有什么区别？假阳性到底是哪个？ ⭐必练"
    **核心答法**：α 是事前定的决策门槛（固定 5%），p-value 是数据算出来的证据（每次不同）。假阳性率在单次检验时 = α。

    | 概念         | 定义                                  | 什么时候定       | 会变吗？         |
    | :----------- | :------------------------------------ | :--------------- | :--------------- |
    | **α**        | 事前设定的假阳性容忍度                | 实验**开始前**   | ❌ 固定           |
    | **p-value**  | H₀ 为真时观察到当前或更极端结果的概率 | 实验**结束后**算 | ✅ 每次不同       |
    | **假阳性率** | 多次实验中 H₀ 为真时错误拒绝的比例    | 由决策规则决定   | ⚠️ Peeking 会膨胀 |

    **类比**：α = 红绿灯规则（p < 0.05 亮绿灯）| p = 卷面分 | 卷面分 < 及格线 → 亮绿灯

    ❌ "假阳性 = 1-p" → 完全错！假阳性率 = α（单次检验场景下）

    !!! tip "面试话术"
        "α 是我事前定的决策门槛，p-value 是数据给我的证据。假阳性率在单次检验时等于 α，但如果反复偷看或做多个检验，整体假阳性率会膨胀超过 α，需要用 Alpha Spending 或 Bonferroni 控制。"

??? warning "Q12: Data Peeking 为什么让假阳性率飙升？怎么算？ ⭐必练"
    **核心逻辑**：每偷看一次 ≈ 多买一张彩票 → 中奖（误判）概率变高。

    **公式**：整体假阳性率 = `1 - (1-α)ᵏ = 1 - 0.95ᵏ`（k = 偷看次数）

    | 偷看次数 | 整体假阳性率 | vs 5%  |
    | :------: | :----------: | :----: |
    |    1     |     5.0%     | 达标 ✅ |
    |    2     |    9.75%     |  ×2 ⚠️  |
    |    5     |    22.6%     | ×4.5 🔴 |
    |    7     |    30.2%     |  ×6 🔴  |
    |    10    |    40.1%     |  ×8 🔴  |

    实际膨胀率比上表略低（因为累积数据有相关性），但面试用 `1 - 0.95ᵏ` 回答已足够。

    **解法** → **Sequential Testing (Alpha Spending)**，见 [A/B 高阶](05a_ab_advanced.md)

??? warning "Q13: 什么是 AA 实验？为什么不直接做 AB？"
    **AA 实验 = 两组都不做干预**，目的是校验实验平台本身是否可信。

    | 方式          | 操作                                  | 优缺点               |
    | :------------ | :------------------------------------ | :------------------- |
    | **预实验 AA** | 先跑 1 周 AA → 不显著 → 保持分流切 AB | ✅ 最严谨，不浪费流量 |
    | **并行三组**  | 对照1:对照2:实验 同时跑               | ⚠️ 浪费 1/3 流量      |
    | **SRM 替代**  | 只跑 AB + SRM 卡方检验                | ✅ 最常用，零额外成本 |

    !!! tip "面试话术"
        "AA 实验就像体检前校准体温计——确认仪器没坏再量病人。实际工作中我用 SRM 诊断替代独立 AA，不占额外流量。但新平台首次上线一定跑完整 AA。"

??? warning "Q14: 效应量 Cohen's d 和 Cohen's h 有什么区别？什么时候用哪个？ ⭐加分题"
    **核心答法**：取决于业务指标是**连续型**还是**比率型**。

    | 效应量指标    | 适用指标类型                  | 统计检验      | 业务例子                          |
    | :------------ | :---------------------------- | :------------ | :-------------------------------- |
    | **Cohen's d** | 连续型指标 (Means)            | T 检验        | 人均订单金额 (ARPU)、人均观看时长 |
    | **Cohen's h** | 比率/概率型指标 (Proportions) | Z 检验 / 卡方 | 转化率 (CVR)、点击率 (CTR)        |

    !!! tip "打磨话术"
        “在评估 A/B 测试或核算样本量时，效应量 (Effect Size) 的选择完全取决于你观测的业务指标。如果是连续型指标（如 ARPU），我会用 Cohen's d（两组均值差除以合并标准差）；如果是互联网最看重的比率型指标（如 CTR），因为底层是二项分布，我会用 Cohen's h（经过反正弦变换的比率差）来衡量差异水平。”
