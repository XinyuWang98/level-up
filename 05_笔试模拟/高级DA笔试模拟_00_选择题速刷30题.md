## 📊 一、统计学基础 (10 题)

### Q1. 假设检验中，p 值的正确含义是？
A. 原假设为真的概率
B. 备择假设为真的概率
C. **在原假设为真的前提下**，观察到当前或更极端结果的概率
D. 实验结果可重复的概率

<details>
<summary>📎 答案与解析</summary>

**答案：C**

p 值是"假设 H0 为真时，观测到当前统计量或更极端值的概率"。**不是** H0 为真的概率（常见错误理解）。面试话术："p 值衡量的是数据与原假设的不兼容程度，而非原假设本身的概率。"
</details>

### Q2. A/B 实验中，当样本量不变时，将显著性水平从 0.05 降到 0.01，会导致什么？
A. 统计功效（Power）增加
B. 统计功效（Power）降低
C. 第一类错误增加
D. 第二类错误降低

<details>
<summary>📎 答案与解析</summary>

**答案：B**

α 降低 → 判断"显著"的门槛更高 → 更难拒绝 H0 → 漏检概率（β）增大 → Power = 1 - β 降低。第一类错误（α）降低了，但代价是第二类错误（β）增大。
</details>

### Q3. 在 A/B 实验中，CUPED 方差缩减的核心原理是？
A. 增大样本量
B. 用实验前数据作为协变量，消除个体差异带来的噪声
C. 降低显著性水平
D. 对数据做标准化处理

<details>
<summary>📎 答案与解析</summary>

**答案：B**

CUPED 公式：$Y_{cuped} = Y - \theta \times (X - E[X])$，其中 X 为实验前的协变量。通过去除个体基线差异（每个人消费习惯不同），大幅降低指标方差，从而用更少样本达到显著。
</details>

### Q4. 某实验 p=0.03，置信区间为 [-2.1, 15.3]，正确的业务解读是？
A. 实验显著，应全量上线
B. 实验显著但置信区间包含负值，提升方向不确定
C. 实验不显著
D. 需要增大样本量

<details>
<summary>📎 答案与解析</summary>

**答案：B**

虽然 p<0.05 显示"统计显著"，但 95% CI 包含负值，说明真实效果可能为负。这可能是检验方法与置信区间方法不完全一致（如单双侧不同），或说明效应非常微弱。**业务决策应该同时看 p 值和置信区间**，不能只看 p。
</details>

### Q5. SRM（Sample Ratio Mismatch）报警表示什么？
A. 指标差异显著
B. 实验组与对照组的样本量比例偏离期望
C. 数据质量问题
D. 实验需要延长周期

<details>
<summary>📎 答案与解析</summary>

**答案：B**

SRM 是指实验分流比例与预期不符（如设计 50:50 但实际 52:48）。使用卡方检验判断偏离程度。出现 SRM 时，实验结果**不可信**，需排查：分流逻辑错误、设备 ID 不稳定、埋点数据丢失、bot 流量。
</details>

### Q6. 辛普森悖论在 A/B 实验中最常见的表现是？
A. 实验组每个子群都优于对照组，但总体不如对照组
B. p 值随时间变化
C. 效应量与样本量正相关
D. 多次实验结果不一致

<details>
<summary>📎 答案与解析</summary>

**答案：A**

经典例子：实验组在男性和女性用户中转化率都高于对照组，但由于男女比例不同（混淆变量），整体转化率对照组反而更高。解决方法：**分层分析**，按关键维度切分后再比较。
</details>

### Q7. 以下哪种情况适合使用 Mann-Whitney U 检验而非 t 检验？
A. 两组数据都服从正态分布
B. 样本量大于 30
C. 数据严重偏态（如订单金额分布）
D. 比较三组以上的差异

<details>
<summary>📎 答案与解析</summary>

**答案：C**

Mann-Whitney U 是非参数检验，不依赖正态假设，适用于偏态数据。t 检验要求数据近似正态（或大样本中心极限定理生效）。订单金额通常是右偏分布，用 U 检验或 Bootstrap 更稳健。
</details>

### Q8. 计算 A/B 实验最小样本量时，以下哪个因素增大会导致所需样本量增加？
A. MDE（最小可检测效应）增大
B. 基线方差增大
C. 显著性水平 α 增大
D. 检验功效 Power 降低

<details>
<summary>📎 答案与解析</summary>

**答案：B**

样本量公式：$n = 2 \times \frac{(Z_{\alpha/2} + Z_\beta)^2 \times \sigma^2}{MDE^2}$

- 方差 σ² 增大 → n 增大 ✅
- MDE 增大 → n 减小（分母变大）
- α 增大 → Z 减小 → n 减小
- Power 降低 → Z_β 减小 → n 减小
</details>

### Q9. 多重比较（同时检验 10 个指标）不做校正的风险是什么？
A. 漏检真实效果
B. 假阳性率膨胀（远超 5%）
C. 实验周期不够
D. 统计功效过高

<details>
<summary>📎 答案与解析</summary>

**答案：B**

检验 10 个指标，每个 α=0.05，至少出现 1 个假阳性的概率 = $1 - (1-0.05)^{10} ≈ 40\%$！远超 5%。解决方法：Bonferroni 修正（α/k）、Benjamini-Hochberg FDR 控制、或预注册核心指标（只对 1-2 个核心指标做检验）。
</details>

### Q10. 因果推断中，DID（双重差分）的核心假设是？
A. 处理组和对照组完全相同
B. 处理组和对照组在无干预情况下趋势平行
C. 数据服从正态分布
D. 样本是随机抽样的

<details>
<summary>📎 答案与解析</summary>

**答案：B**

平行趋势假设（Parallel Trends Assumption）：在没有干预的情况下，处理组和对照组的指标变化趋势应该是相同的。验证方法：画出干预前多期的趋势线，检查是否平行。这是 DID 最关键也最常被面试追问的假设。
</details>

### Q11. 以下 SQL 的 `COUNT(DISTINCT ...)` 和 `COUNT(...)` 有何区别？
```sql
SELECT COUNT(user_id), COUNT(DISTINCT user_id) FROM orders;
-- 假设 orders 有 1000 行，其中 300 个不同用户
```
A. 两者结果相同
B. COUNT=1000, COUNT(DISTINCT)=300
C. COUNT=300, COUNT(DISTINCT)=1000
D. 两者都等于 300

<details>
<summary>📎 答案与解析</summary>

**答案：B**

`COUNT(user_id)` 统计所有非 NULL 的行数（1000 行）。`COUNT(DISTINCT user_id)` 统计去重后的唯一值数（300 个用户）。这是最基础但最常考的 SQL 区别。
</details>

### Q12. 以下 SQL 中 `LEFT JOIN` 和 `INNER JOIN` 的区别是什么？
```sql
-- 表A有100个用户，表B有80个用户有订单
SELECT * FROM users a LEFT JOIN orders b ON a.id = b.user_id;
SELECT * FROM users a INNER JOIN orders b ON a.id = b.user_id;
```
A. LEFT JOIN 返回 80 行，INNER JOIN 返回 100 行
B. LEFT JOIN 返回 100+ 行（含无订单用户），INNER JOIN 仅返回有订单的行
C. 两者结果相同
D. LEFT JOIN 会返回笛卡尔积

<details>
<summary>📎 答案与解析</summary>

**答案：B**

LEFT JOIN 保留左表（users）所有行，即使右表没有匹配。INNER JOIN 只返回两表都有匹配的行。注意：如果一个用户有多个订单，LEFT JOIN 和 INNER JOIN 都会产生多行（一对多关系）。
</details>

### Q13. `WHERE` 和 `HAVING` 的根本区别是？
A. 没有区别，可以互换
B. `WHERE` 在分组前过滤，`HAVING` 在分组后过滤
C. `HAVING` 只能用于数值列
D. `WHERE` 不能用于聚合函数

<details>
<summary>📎 答案与解析</summary>

**答案：B（和 D）**

SQL 执行顺序：FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY。`WHERE` 在 `GROUP BY` 之前执行，不能使用聚合函数；`HAVING` 在 `GROUP BY` 之后执行，可以使用聚合函数（如 `HAVING COUNT(*) > 5`）。
</details>

### Q14. 以下窗口函数 `RANK()` vs `DENSE_RANK()` vs `ROW_NUMBER()` 对并列值的处理？
```sql
-- 分数: 100, 100, 90, 80
```
A. RANK: 1,1,2,3 / DENSE_RANK: 1,1,3,4 / ROW_NUMBER: 1,2,3,4
B. RANK: 1,1,3,4 / DENSE_RANK: 1,1,2,3 / ROW_NUMBER: 1,2,3,4
C. 三者结果相同
D. RANK: 1,2,3,4 / DENSE_RANK: 1,1,2,3 / ROW_NUMBER: 1,1,3,4

<details>
<summary>📎 答案与解析</summary>

**答案：B**

- `RANK()`：并列后**跳过**名次 → 1,1,**3**,4
- `DENSE_RANK()`：并列后**不跳过** → 1,1,**2**,3
- `ROW_NUMBER()`：无视并列，严格递增 → 1,2,3,4

面试必考三兄弟！最常用的是 ROW_NUMBER（取 Top N 时不会被并列干扰）。
</details>

### Q15. NULL 值在 SQL 中的行为？
```sql
SELECT 1 = NULL, 1 <> NULL, NULL = NULL, NULL IS NULL;
```
A. FALSE, TRUE, TRUE, TRUE
B. NULL, NULL, NULL, TRUE
C. FALSE, FALSE, FALSE, TRUE
D. NULL, NULL, TRUE, TRUE

<details>
<summary>📎 答案与解析</summary>

**答案：B**

NULL 与任何值比较都返回 NULL（不是 TRUE/FALSE）。只有 `IS NULL` / `IS NOT NULL` 才能正确判断 NULL。这就是为什么 `WHERE column = NULL` 永远选不出任何行，必须写 `WHERE column IS NULL`。
</details>

### Q16. 以下 SQL 的 `GROUP BY` 结果有几行？
```sql
SELECT city, gender, COUNT(*) 
FROM users 
GROUP BY city, gender;
-- 假设有 5 个城市、2 种性别
```
A. 5 行
B. 2 行
C. 10 行（5×2）
D. 取决于数据

<details>
<summary>📎 答案与解析</summary>

**答案：D**

`GROUP BY city, gender` 按 (city, gender) 的唯一组合分组。如果每个城市都有男女两种性别，最多 10 行；但如果某城市只有男性用户，则该城市只有 1 行。答案取决于实际数据中存在的组合数。
</details>

### Q17. `UNION` 和 `UNION ALL` 的区别？
A. `UNION` 去重，`UNION ALL` 不去重
B. `UNION ALL` 去重，`UNION` 不去重
C. `UNION` 更快
D. 没有区别

<details>
<summary>📎 答案与解析</summary>

**答案：A**

`UNION` 对合并结果进行去重（Extra: Using temporary; Using filesort），性能较差。`UNION ALL` 直接合并不去重，性能更好。**实际工作中 99% 的场景应该用 `UNION ALL`**，除非你明确需要去重。
</details>

### Q18. `LAG()` 和 `LEAD()` 窗口函数的方向？
```sql
SELECT dt, amount,
    LAG(amount, 1) OVER (ORDER BY dt),   -- ?
    LEAD(amount, 1) OVER (ORDER BY dt)   -- ?
FROM daily_sales;
```
A. LAG 取后一行，LEAD 取前一行
B. LAG 取前一行，LEAD 取后一行
C. 两者都取当前行
D. LAG 取最大值，LEAD 取最小值

<details>
<summary>📎 答案与解析</summary>

**答案：B**

`LAG(col, n)` = 取当前行**前** n 行的值（滞后/回顾）。`LEAD(col, n)` = 取当前行**后** n 行的值（超前/预测）。常用场景：计算环比（`amount - LAG(amount,1)`）、日期差。
</details>

### Q19. 子查询 vs CTE（WITH 子句），以下说法正确的是？
A. CTE 性能一定比子查询好
B. CTE 可以递归，子查询不行
C. 子查询可以在 SELECT 中使用，CTE 不行
D. 两者完全等价，只是写法不同

<details>
<summary>📎 答案与解析</summary>

**答案：B**

CTE 支持递归查询（`WITH RECURSIVE`），子查询不行。性能方面，大部分数据库优化器对两者处理相同。CTE 的主要优势是**可读性**（特别是多层嵌套时），以及在同一查询中**可复用**。面试建议：**永远用 CTE 写，可读性加分**。
</details>

### Q21. 抛一枚均匀硬币 10 次，恰好出现 5 次正面的概率最接近？
A. 50%
B. 25%
C. 10%
D. 5%

<details>
<summary>📎 答案与解析</summary>

**答案：B**

二项分布：$P(X=5) = C(10,5) \times 0.5^{10} = 252/1024 ≈ 24.6\%$。虽然期望是 5 次，但恰好是 5 次的概率只有约 25%，不是 50%！这是概率直觉陷阱。
</details>

### Q22. 一个分类模型的 Precision=0.9, Recall=0.6，以下说法正确的是？
A. 模型预测为正的样本中 90% 确实是正的，但 40% 的正样本被漏掉了
B. 模型 60% 的预测是正确的
C. 模型准确率是 90%
D. F1 分数 > 0.9

<details>
<summary>📎 答案与解析</summary>

**答案：A**

Precision=0.9：预测为正的中 90% 是对的（误报少）。Recall=0.6：所有真正的正样本中只抓到了 60%（漏检多）。F1 = 2×0.9×0.6/(0.9+0.6) = 0.72 < 0.9。这是"宁可漏检也不误报"的保守模型。
</details>

### Q23. 某电商平台转化率从 2% 提升到 2.2%，提升幅度描述最准确的是？
A. 提升了 0.2%
B. 提升了 10%
C. 提升了 0.2 个百分点（绝对值），10%（相对值）
D. 提升了 20%

<details>
<summary>📎 答案与解析</summary>

**答案：C**

绝对提升 = 2.2% - 2% = 0.2pp（百分点）。相对提升 = (2.2-2)/2 = 10%。面试中混淆绝对值和相对值是大忌！正确说法："转化率提升了 0.2 个百分点，相对提升 10%。"
</details>

### Q24. 贝叶斯公式 P(A|B) = P(B|A)×P(A)/P(B) 的直观含义是？
A. 条件概率的逆运算
B. 用新证据（B发生了）更新对 A 的信念
C. A 和 B 独立
D. 联合概率的分解

<details>
<summary>📎 答案与解析</summary>

**答案：B**

P(A) 是先验概率（你对 A 的初始信念），P(B|A) 是似然（如果 A 为真，看到 B 的概率），P(A|B) 是后验概率（看到 B 之后对 A 的更新信念）。业务应用：垃圾邮件过滤、异常检测。
</details>

### Q26. 中心极限定理（CLT）的核心含义是？
A. 所有数据都服从正态分布
B. 样本均值的分布随样本量增大趋近正态分布
C. 大样本量能消除偏差
D. 方差随样本量减小

<details>
<summary>📎 答案与解析</summary>

**答案：B**

无论总体分布是什么形状（偏态、均匀、双峰...），**样本均值的抽样分布**在样本量足够大时趋近正态分布。这是 t 检验和 z 检验能工作的数学基础。一般 n ≥ 30 被认为"足够大"。
</details>

### Q28. RFM 模型中的三个维度分别是？

<details>

### Q30. A/B 实验中，"新奇效应（Novelty Effect）"是指？
A. 新功能长期效果比短期更好
B. 用户因为好奇心对新功能短期反应过度，长期效果回落
C. 实验组和对照组没有差异
D. 样本量不足导致的虚假显著
