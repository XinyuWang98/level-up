# 📄 简历（CV）

> 存放你的 Markdown 格式简历，用 Agent 的 `/generate_resume` workflow 进行定制化优化

## 📖 这个模块做什么？

在这里放置你的简历文件。推荐使用 Markdown 格式，方便 Agent 读取和修改。系统支持两种简历策略：

1. **通用版简历** — 一份涵盖所有核心经历的基础版本
2. **定制版简历** — 针对特定 JD 由 Agent 优化的版本

## 📂 文件说明

| 文件                           | 说明                          |
| :----------------------------- | :---------------------------- |
| `DA_5YOE_Example.md` | Mock 简历示例（展示推荐格式） |

## 🚀 如何使用？

### Step 1: 创建你的通用版简历

参考 `DA_5YOE_Example.md` 的格式，创建你自己的简历：

```markdown
# 你的名字

## 联系方式
- Email: xxx
- GitHub: xxx

## 职业概要
一句话概括你的核心竞争力

## 工作经历
### [公司名] | [职位] | [时间]
**项目 1: [项目名]**
- **S(Situation)**: 背景
- **T(Task)**: 任务
- **A(Action)**: 你做了什么
- **R(Result)**: 量化结果

## 技能
- Python (Pandas, NumPy, Scikit-learn)
- SQL (复杂查询, 窗口函数)
- ...
```

### Step 2: 根据 JD 定制

```
/generate_resume
请根据以下 JD 优化我的简历，重点突出匹配的技能和项目。
[粘贴 JD]
```

## 💡 来自作者的经验

- **用 STAR 法则写项目**：面试官看简历的第一眼就在找 Result（量化成果）
- **通用版只改措辞重心**：核心经历不要为了不同 JD 大改，只需要调整强调的能力维度
- **简历不超过 2 页**：招聘方平均看一份简历的时间不超过 30 秒
